# Local API 快速开始

语言版本：
[English](../English/quickstart.md)

相关文档：
[总览](./总览.md) · [接口参考](./接口参考.md) · [插件开发指南](./插件开发指南.md)

## 1. 在 Lattice 中启用 Local API

打开：

`Settings → General → Local API`

可以看到以下配置项：

- `Enabled`
- `Port`
- `Base URL`

默认端口是 `52731`，可配置范围是 `1024` 到 `65535`。

常见默认地址：

```text
http://127.0.0.1:52731
```

API 前缀：

```text
http://127.0.0.1:52731/api/v1
```

如果你修改了端口，现有客户端需要重新连接到新地址。

## 2. 先做健康检查

```bash
curl http://127.0.0.1:52731/api/v1/status
```

示例响应：

```json
{
  "ok": true,
  "apiVersion": "1",
  "appVersion": "1.0.0 (123)",
  "capabilities": [
    "search",
    "paper-detail",
    "csl-item",
    "plugin-hosting"
  ]
}
```

说明：

- `ok` 为 `true` 代表服务可访问
- `apiVersion` 是当前 HTTP API 版本
- `appVersion` 是当前 Lattice 应用版本
- `capabilities` 是当前实例支持的功能列表

## 3. 试一次搜索

```bash
curl "http://127.0.0.1:52731/api/v1/search?q=graph%20neural%20network&limit=5"
```

如果 `q` 为空，接口会返回最近加入文献库的条目。

## 4. 读取单篇文献详情

拿到搜索结果中的 `id` 之后：

```bash
curl http://127.0.0.1:52731/api/v1/papers/550E8400-E29B-41D4-A716-446655440000
```

该接口返回面向引用场景的完整文献快照，同时内嵌 `cslItem`，可直接交给 CSL 处理器。

## 5. 典型接入方式

### 方式 A：脚本或本地服务

直接请求完整 URL：

```js
const baseURL = "http://127.0.0.1:52731/api/v1";
const status = await fetch(`${baseURL}/status`).then((r) => r.json());
```

### 方式 B：插件页面

如果你的前端页面由 Lattice 自己托管在 `/plugins/{name}/...` 下，推荐使用相对路径：

```js
const response = await fetch("/api/v1/status");
```

这样插件页面和 API 天然同源，不需要自己处理跨域问题。

## 6. 安全与访问限制

Local API 的设计目标是“本机集成”，不是公网服务。

- 只监听本机回环地址
- 默认只允许本地浏览器来源访问
- 仅支持 `GET` 和 `OPTIONS`
- 不提供写接口

这意味着：

- 远程机器无法直接访问这套 API
- 普通浏览器页面如果不在本地来源上，默认不能直接跨域访问
- 最稳妥的 Web 集成方式是直接使用 `/plugins/{name}/...` 托管插件页面

## 7. 遇到连接不上时先检查

1. Lattice 是否正在运行
2. `Enabled` 是否打开
3. 端口是否已经改动
4. 你的请求地址是不是仍指向旧端口
5. 如果你的宿主集成使用了 manifest、入口 URL 缓存或安装脚本，修改端口后是否已经重新生成或重新安装
