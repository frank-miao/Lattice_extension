# Lattice Local API

语言版本：
[English](../English/README.md)

Lattice 提供了一个只在本机监听的、只读的 HTTP API，方便外部工具访问文献库中的检索结果与引文元数据，而不需要直接依赖 Lattice 的内部数据模型或应用包。

相关文档：
[快速开始](./快速开始.md) · [接口参考](./接口参考.md) · [插件开发指南](./插件开发指南.md)

这套文档面向外部开发者，重点回答下面几类问题：

- Local API 能做什么
- 提供了哪些端点
- 返回的数据结构是什么
- 怎样开发自己的本地插件或集成

## 能力概览

| 能力 | 说明 |
| --- | --- |
| 本地健康检查 | 检查 Local API 是否可用、当前 API 版本、Lattice 版本、能力列表 |
| 文献搜索 | 按标题、作者、期刊/出版源、citekey、年份搜索文献 |
| 单篇文献详情 | 获取单篇文献的详细引用元数据 |
| CSL-JSON 输出 | 在文献详情中直接返回可供 citeproc 等处理器使用的 `cslItem` |
| 插件静态资源托管 | 通过 `/plugins/{name}/...` 托管本地插件前端资源 |

## 设计边界

- 这是一个只读 API，不提供写入、修改、删除接口。
- 这是一个本地 API，只监听 `127.0.0.1`。
- 面向本机脚本、自动化工具、桌面伴生工具、Office/Obsidian 一类插件式集成。
- 所有业务端点都位于 `/api/v1` 之下。

## 一分钟上手

假设你的 Local API 运行在默认端口 `52731`：

```bash
curl http://127.0.0.1:52731/api/v1/status
curl "http://127.0.0.1:52731/api/v1/search?q=transformer&limit=5"
curl http://127.0.0.1:52731/api/v1/papers/550E8400-E29B-41D4-A716-446655440000
```

## 适合怎样的集成

- Shell / Python / Node 脚本：直接请求 `http://127.0.0.1:<port>/api/v1/...`
- 本地 Web UI：将静态资源部署到 `/plugins/{name}/...`，再通过相对路径请求 `/api/v1/...`
- Office / 桌面插件：让宿主加载 `/plugins/{name}/...` 下的页面，再由页面调用 Local API

## 现有外部扩展参考

仓库中的 `Lattice_plugins/` 目录包含基于 Local API 实现的外部扩展资源，可作为插件打包方式、静态资源组织方式以及本地托管接入方式的参考。
